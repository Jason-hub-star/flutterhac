# Track Strategy

해커톤 트랙은 셋 다 노리되, 한 번에 다 잘하려고 하면 산만해지기 쉽다. 기본 전략은 `AI Harness`를 주력으로 잡고, `Widget Craft`를 심사위원이 바로 체감하는 표면으로 삼고, `Wild Card`는 한 방의 인상으로 설계하는 것이다.

## Track Summary

### 1. AI Harness

가장 잘 밀어붙인 팀으로 보이려면 아래 3개가 보여야 한다.

- 실제 사용자가 결과를 얻기까지의 반복을 줄인다.
- 프롬프트나 에이전트가 단순 채팅이 아니라 UI나 작업 흐름을 실질적으로 움직인다.
- 실패했을 때도 망가지지 않는 fallback이 있다.

증빙으로 좋은 것:

- 구조화된 입력/출력
- 자동화 전후 비교
- 한 번의 요청으로 여러 단계를 줄인 데모
- 도구 호출 또는 상태 전이 로그

### 2. Widget Craft

완성도가 돋보이려면 "예쁜 화면"만으로는 부족하다.

- 정보 구조가 즉시 이해된다.
- 디테일한 상태 변화가 자연스럽다.
- 폰트, 여백, 컬러, 컴포넌트 위계가 일관된다.
- 빈 상태, 로딩, 실패 상태까지 설계돼 있다.

증빙으로 좋은 것:

- 첫 화면과 핵심 액션 화면
- 모바일 실제 기기 캡처
- 다크/라이트보다 정보 위계와 상호작용의 설득력

### 3. Wild Card

이 트랙은 전체 앱 본체와 분리된 놀라운 장면이 필요하다.

- 예상 밖의 입력 방식
- 에이전트가 UI를 재구성하는 장면
- 영상, 음성, OCR, 멀티모달, 실시간 피드백 같은 비정형 경험

핵심은 "창의적이지만 앱의 문제 해결과 연결돼야 한다"는 점이다.

## Recommended Positioning

### Default

- 주력: `AI Harness`
- 보조: `Widget Craft`
- 차별화: `Wild Card`

### Why This Mix Works

- AI Harness는 짧은 시간에도 이야기거리가 강하다.
- Widget Craft는 심사위원이 앱을 보는 첫 5초를 책임진다.
- Wild Card는 전체 점수를 끌어올리는 기억 포인트가 된다.

## Scoring Narrative Template

아래 문장을 프로젝트에 맞게 바꿔서 발표/문서에 일관되게 사용한다.

- `우리 팀은 AI를 넣은 앱이 아니라, AI가 실제 사용자 흐름을 줄여주는 제품을 만들었다.`
- `이 앱은 첫 화면부터 완성도가 느껴지도록 핵심 행동을 하나의 강한 표면으로 압축했다.`
- `그리고 마지막으로, 경쟁작과 구분되는 창의적 인터랙션을 한 장면으로 남긴다.`

## Proof Matrix

### AI Harness

- [ ] 자동화 대상이 명확하다
- [ ] 자동화 전보다 입력 횟수나 시간 절약이 설명된다
- [ ] 실제 작동 예시가 있다
- [ ] 실패 시 fallback이 있다

### Widget Craft

- [ ] 대표 화면 1개가 강하다
- [ ] 로딩/빈 상태/에러 상태가 있다
- [ ] 타이포와 여백이 정리돼 있다
- [ ] 발표 기기에서 깨지지 않는다

### Wild Card

- [ ] 한 문장으로 설명 가능한 창의 포인트가 있다
- [ ] 본체와 분리된 안전한 실험 구조다
- [ ] 짧은 데모로 와우 포인트를 전달할 수 있다

## Anti-Patterns

- AI를 넣었지만 실제 사용 흐름이 전혀 줄지 않는 경우
- 위젯은 많지만 정보 구조가 산만한 경우
- Wild Card를 위해 본체 완성도를 희생하는 경우
- 설치/로그인/API 키 문제로 데모가 멈추는 경우

## Final Advice

해커톤에서는 기능 개수보다 "심사위원이 기억할 장면 2개"가 더 중요하다.

- 장면 1: AI가 실제 작업을 줄여주는 순간
- 장면 2: 앱이 의외로 완성도 높고 인상적인 순간

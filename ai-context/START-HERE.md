# Start Here

## Purpose

이 저장소의 1차 목적은 Flutter 해커톤 프로젝트를 빠르게 완성 가능한 상태로 끌고 가는 것이다.

## What Exists

- 프로젝트 완성 명세서 템플릿
- 3개 트랙 대응 전략 문서
- Flutter 공식 문서 및 대표 오픈소스 레퍼런스
- 맥북/윈도우/깃허브/Flutter 초기 세팅 가이드
- 재사용 가능한 Codex 스킬

## Current Assumptions

- 주 개발자는 MacBook을 사용한다.
- 팀원은 Windows 노트북을 사용한다.
- VS Code 앱은 설치돼 있을 수 있지만 `code` CLI는 아직 확인되지 않았다.
- `flutter`와 `dart`는 아직 설치되지 않은 상태다.
- 원격 GitHub 저장소는 비어 있었고, 현재 로컬 작업 트리와 연결되었다.

## First Sequence

1. `docs/ref/TRACK-STRATEGY.md`에서 주력 트랙을 정한다.
2. `docs/specs/PROJECT-COMPLETION-SPEC-TEMPLATE.md`를 복제해 실제 프로젝트용 명세서를 작성한다.
3. `docs/setup/BOOTSTRAP-SETUP.md`에 따라 MacBook과 Windows 개발 환경을 연다.
4. 구현이 시작되면 `docs/status/PROJECT-STATUS.md`와 `docs/status/EXECUTION-CHECKLIST.md`를 갱신한다.
5. Codex에게 반복적으로 맡길 때는 `skills/flutter-hackathon-bootstrap/SKILL.md`를 사용한다.

## Recommended Default

- 주력 트랙: `AI Harness`
- 보조 트랙: `Widget Craft`
- 차별화 장치: `Wild Card`를 위한 한 가지 놀라운 상호작용 또는 데모 연출

이 기본값은 시간이 짧을수록 유리하다. 완전한 기능 수보다 "자동화 데모가 확실히 먹히는 제품"을 우선한다.
